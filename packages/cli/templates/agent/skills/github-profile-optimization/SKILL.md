# GitHub profile optimization

Make the profile **sell the target role** in under 30 seconds of scrolling.

## When to use

- User runs `/improve` or asks about bio, pins, README, portfolio presentation.
- Fit score is decent but profile looks unfocused (mixed stacks, empty bio, weak pins).
- User prepares for recruiters or OSS visibility.

## Workflow

1. Read **target role**, **primary stack**, and **top 3 strength repos** from profile data.
2. Audit four surfaces:
   - **Bio** — role + stack + one proof point (max ~160 chars mindset)
   - **Pinned repos** — do pins match target role? suggest reorder or replacements
   - **Profile README** — missing? suggest structure (who, stack, featured repos, contact)
   - **Repo READMEs** — flag showcase repos with thin or missing READMEs
3. Use `/improve` output when available; expand with repo-specific examples.
4. End with **3 prioritized edits** ranked by hiring-manager impact (pins > bio > profile README).

## Output format

**Profile audit**
- Bio: …
- Pins: …
- README: …

**Top 3 changes**
1. …
2. …
3. …

Do not suggest fake metrics, bought stars, or misrepresenting repo ownership.
