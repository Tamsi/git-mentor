# Interview prep from GitHub

Turn public repos into **talking points** a candidate can use in technical and behavioral interviews.

## When to use

- User has an interview or wants to "present my GitHub" to a hiring manager.
- User asks what to highlight, demo, or explain about their work.

## Workflow

1. Select **3 strength repos** with the strongest **role alignment** and clearest evidence (languages, manifests, stars, topics).
2. For each repo, write a **60-second pitch**:
   - Problem / scope (1 sentence)
   - Stack & your contribution signal (from repo metadata — be honest if solo OSS)
   - Outcome or learning (what a reviewer should take away)
3. Prepare **one gap question**: use gap analysis to pick a weakness interviewers may probe; give an honest answer + mitigation (course, WIP repo, contribution plan).
4. Recommend **2 repos to demo live** (README quality, runnable signals from manifests) and **1 to link** without demo.
5. Optional: suggest `/analyze <repo>` on the demo repo if manifest details would strengthen the story.

## Output format

**Highlight repos**
1. `owner/repo` — pitch …
2. …
3. …

**Prepare for:** "…" → …

**Demo:** … | **Link:** …

Never claim team size, production scale, or business impact not supported by public data.
